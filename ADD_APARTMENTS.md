# Добавление новых квартир в проект

Пошаговая инструкция для добавления новых объявлений с Циан: что сделать вам и что нужно сообщить ассистенту (или сделать в коде), чтобы всё работало — парсинг, фото, карта и **точные координаты** на карте.

---

## 1. Откуда берутся новые квартиры

Два варианта.

### Вариант А: У вас есть сохранённая страница «Избранное»

1. На Циан откройте «Избранное» с нужными объявлениями.
2. Сохраните страницу: **Файл → Сохранить как → Веб-страница, полностью**.
3. Сохраните файл куда удобно (например `data/favorite_3.html`).
4. Создайте новую папку под статику, например **`data/static_4/`** (если уже есть `static_2` и `static_3`, следующий номер — `static_4`).
5. В эту папку положите для **каждой** новой квартиры:
   - файл **`<ID>.html`** — сохранённая страница объявления (ID из ссылки, например `https://spb.cian.ru/sale/flat/326727470` → `326727470.html`);
   - папку **`<ID>_files/`** — только нужные картинки объявления (лишние удалите), имена можно не менять.

Итого в `data/static_4/` лежит, например:
- `326727470.html`
- `326727470_files/` (фото)
- `326732626.html`
- `326732626_files/` (фото)
- …

### Вариант Б: Нет сохранённой страницы — только HTML и папка со статикой

1. Вы присылаете ассистенту **HTML-код карточек** новых объявлений (например, скопированный из страницы избранного или списка).
2. Отдельно у вас уже готова **новая папка статики** (например `data/static_4/`) с:
   - `<ID>.html` — страницы объявлений (если есть);
   - `<ID>_files/` — только нужные фото по каждому ID, лишние файлы удалены.

Ассистент по HTML извлечёт ссылки, адреса, цену, площадь, метро и т.д. и добавит записи в `data/apartments.json`, затем настроит парсер под новую папку (`static_4`) и подтянет фото/описание из HTML.

---

## 2. Что нужно сделать в проекте при новой папке (static_4, static_5, …)

Чтобы новые квартиры из новой папки отображались на карте с фото и описанием, делается следующее.

### Шаг 1: Добавить поддержку новой папки в парсере

В **`scripts/parse_cian_offer_pages.py`**:

1. Добавить константу, например:
   ```python
   STATIC_4_DIR = os.path.join(ROOT, 'data', 'static_4')
   ```
2. Добавить функцию сбора фото, по аналогии с `collect_local_photos_static3`:
   ```python
   def collect_local_photos_static4(offer_id: str):
       folder = os.path.join(STATIC_4_DIR, offer_id + '_files')
       ...
       rel = 'data/static_4/' + offer_id + '_files/' + ...
   ```
3. В `main()`:
   - сканировать `STATIC_4_DIR` в поисках `(\d+)\.html`;
   - для каждого такого ID записывать в `files_by_id` путь к HTML и источник `'static_4'`;
   - в цикле обработки при `source == 'static_4'` вызывать `collect_local_photos_static4(offer_id)` и подставлять фото из `static_4`.

То же повторяется для `static_5`, `static_6` и т.д. по мере появления новых папок.

### Шаг 2: Добавить записи по новым квартирам в `data/apartments.json`

Для каждой новой квартиры в конец массива в **`data/apartments.json`** добавляется объект с полями как у существующих:

- `url` — ссылка на объявление на Циан;
- `title`, `price`, `address`, `metro`, `phone`, `description` — по возможности из HTML или заглушки;
- `img_src` — пустая строка или первый путь к фото (парсер перезапишет);
- `photos` — пустой массив (парсер заполнит из папки `<ID>_files/`);
- **`lat`, `lon`** — **временно** можно поставить приблизительные (например центр района); затем их нужно заменить на точные (см. шаг 3);
- `price_value`, `total_area`, `floor`, `price_per_sqm`, `build_year` — по данным из HTML или `null`.

После добавления записей запускается парсер — он подтянет описание, цену, площадь, этаж, год постройки и список фото из соответствующих HTML и папок `*_files/`.

### Шаг 3: Подставить точные координаты для новых квартир

**Обязательно:** для каждой новой квартиры нужно подобрать **точные координаты** по адресу, иначе маркер на карте окажется в парке, на канале или в другом районе.

Рекомендуется:

1. Найти адрес в **2ГИС** (например «Ленинский проспект 54 Санкт-Петербург» или «проспект Героев 18 СПб»).
2. Открыть карточку дома и взять координаты из URL маршрута или подписи на карте (формат обычно `lon,lat`, например `30.16896,59.863282` → в JSON: `"lat": 59.863282, "lon": 30.16896`).
3. В **`data/apartments.json`** в соответствующей записи заменить `lat` и `lon` на найденные значения.
4. Перегенерировать карту:
   `python scripts/create_map_cian.py`

Повторить для всех новых квартир. Эту проверку координат нужно делать **каждый раз** при добавлении новых объявлений.

### Шаг 4: Добавить новую папку в `.gitignore` (чтобы в git попадали только картинки)

Чтобы поведение было таким же, как у `static_2` и `static_3`: сама папка в репозиторий не коммитится целиком, но **картинки .jpg и .jpeg** из подпапок `<ID>_files/` — **попадают**. Так на GitHub Pages отображаются превью без загрузки HTML и лишних файлов.

В **`.gitignore`** добавьте блок по аналогии с `static_3`:

```
!/data/static_4/
/data/static_4/*
!/data/static_4/*_files/
/data/static_4/*_files/*
!/data/static_4/*_files/*.jpg
!/data/static_4/*_files/*.jpeg
```

Итог: в git не попадают HTML и прочее содержимое `static_4`, в репозиторий добавляются только файлы `data/static_4/<ID>_files/*.jpg` и `*.jpeg`.

**Как устроен `.gitignore` для static-папок:** папки `data/static_2/` и `data/static_3/` в целом игнорируются (`/data/static_N/*`), но через правила-исключения (`!...`) в репозиторий разрешены только файлы `data/static_N/<ID>_files/*.jpg` и `*.jpeg`. Остальное (HTML, другие расширения) в git не попадает. Для каждой новой папки (`static_4`, `static_5` и т.д.) добавляется такой же блок правил.

---

## 3. Краткий чек-лист при добавлении новых квартир

- [ ] Создана папка `data/static_N/` и в неё положены `<ID>.html` и `<ID>_files/` с фото.
- [ ] В `parse_cian_offer_pages.py` добавлены `STATIC_N_DIR`, `collect_local_photos_staticN` и учёт `static_N` в `main()`.
- [ ] В `data/apartments.json` добавлены объекты по каждой новой квартире (в т.ч. временные `lat`/`lon`).
- [ ] Запущен парсер: `python scripts/parse_cian_offer_pages.py`.
- [ ] Для каждой новой квартиры подобраны **точные координаты** (2ГИС и т.п.) и обновлены `lat`/`lon` в `apartments.json`.
- [ ] Перегенерирован JS: `python scripts/create_map_cian.py`.
- [ ] При необходимости в `.gitignore` добавлены правила для `data/static_N/` (чтобы в git попадали только картинки .jpg/.jpeg из `*_files/`).

После этого новые квартиры появятся на карте в правильных точках, с фото и описанием из новой папки.

---

## 4. Ссылки на скрипты

- **Парсер страниц объявлений и фото:** `scripts/parse_cian_offer_pages.py`
  Читает HTML из `data/`, `data/static_2/`, `data/static_3/` (и при необходимости из новых `static_N/`), обновляет `data/apartments.json`.
- **Генерация карты:** `scripts/create_map_cian.py`
  Строит `data/apartments.js` из `data/apartments.json` (в т.ч. с учётом координат и разброса при дубликатах).

Подробнее про быстрый старт и публикацию на GitHub Pages — в [README.md](README.md).
